# Parcel Semantics

The _Parcel_ language semantics comprise the _export_ and _import_ `#include` files that express the `parcel` encapsulation. The following sections specify the generated file contents for exported _typedef_, _variable_, and _function_ identifiers. For an overview of the translation process and its diagnostics, see [TRANSLATION.md](TRANSLATION.md).

## Typedefs

### Canonical Name

Each exported identifier is assigned a globally unique _canonical name_ derived from its export path, parcel name, and identifier:

```
<path>/<name> { <identifier> }  →  <path_underscored>_<name>_<identifier>
```

Each `/` in the path is replaced with `_`. For example, a parcel declared as `p` at path `foo` exporting identifier `T` produces the canonical name `foo_p_T`.

The canonical name is the stable cross-file identity of the exported identifier. Export files introduce it as an alias for the local definition; import files introduce a stem-prefixed alias for it. This two-step indirection means import files depend only on the canonical name, not on the declaring file directly.

### Export

For each typedef identifier `<Id>` in a `#pragma parcel <name> { ... }` declaration, the translator generates one line in `export/<path>/<name>`:

```c
typedef <Id> <canonical>;
```

where `<canonical>` is the canonical name for that identifier. The export file must be included _after_ all typedef definitions it references, so that `<Id>` is already defined when the preprocessor expands the include.

### Import

For each typedef identifier `<Id>` in the named parcel, the translator generates one line in `import/<path>/<name>.<stem>`:

```c
typedef <canonical> <stem>_<Id>;
```

The importer uses `<stem>_<Id>` as the local name throughout the file. The stem is chosen freely by the importer and scopes the imported names to prevent collisions.

### Example

**`foo.c`** — declares and exports parcel `p` at path `foo`, exporting typedef `T`:

```c
#pragma parcel p { T }

typedef int T;

#include "export/foo/p"
```

**`_parcel/export/foo/p`** — generated by the translator:

```c
typedef T foo_p_T;
```

Including this after `typedef int T;` makes `foo_p_T` an alias for `int` within `foo.c` and establishes the canonical name for importers.

**`bar.c`** — imports parcel `foo/p` with stem `s`:

```c
#include "import/foo/p.s"

s_T v;
```

**`_parcel/import/foo/p.s`** — generated by the translator:

```c
typedef foo_p_T s_T;
```

`s_T` in `bar.c` names the same type as `T` in `foo.c`. The stem `s` is local to `bar.c`; a different file importing the same parcel may use a different stem without affecting the type identity.
